# Security audit — pqcscan v0.6.9

**Date:** 2026-06-22
**Scope:** daemon/web surface, probe subprocess paths, parsers, file/path handling, secret exposure, supply chain, and the shipped production deploy posture (systemd unit + `docs/DEPLOYMENT.md`).
**Method:** multi-agent audit — attack-surface map (5 agents) → 9 specialist reviewers → adversarial verification (3 independent skeptics per finding across exploitability / reachability / correctness lenses; kept only when ≥2 confirmed) → synthesis. 96 agents total.
**Result:** 27 findings raised, **14 survived verification**, **12** in this deduped report. **Overall risk: HIGH.**

> Verification was **static** (code-read), not dynamic. See *Coverage gaps* — notably probe command-injection needs a dedicated pass.

## Dominant theme — privilege amplification

The daemon is a **root-privileged, unauthenticated FastAPI app** (under the shipped systemd unit) whose entire security model is a `127.0.0.1` bind. A loopback bind is **not** an auth boundary: any unprivileged local user, co-tenant process, **or a malicious browser tab the operator visits** (no CORS / Origin / CSRF / Host filtering anywhere — `app.py:19-222`) reaches the full API. Every finding below inherits that reachability.

## Top remediations (in order)

1. **Real auth boundary** — bearer token / session on all data-bearing + state-changing endpoints; `--bind` fails closed on any non-loopback address unless a token is set; add `TrustedHostMiddleware` + Origin allowlist. *This single change downgrades most other findings.*
2. **Stop leaking evidence** — gate report/export behind auth **and** redact absolute paths to basenames in the technical report and XLSX evidence.
3. **Lock the DB** — `chmod` the `.db`/`-wal`/`-shm` to `0600` and parent to `0700` in `Repo.__init__` + CLI mkdir paths, independent of umask/systemd.
4. **Fix the supply chain** — pin an exact syft version + verify its sha256/signature (kill `curl | sh`); add a `SHA256SUMS` asset + build-provenance attestation to `release.yml`.
5. **Bound the resource vectors** — single-flight + 409 on `POST /api/scans`; bounded SSE queue with slow-consumer drop; route all XLSX cell writes through a formula-injection-safe helper; HTML-escape the SSE payload now to defuse L3.

## Affects artifacts shipped this session

- **H1/H6** are amplified by the **root systemd unit** I added (`packaging/systemd/pqcscan.service`). The `DynamicUser=yes` variant already documented reduces blast radius; given this audit, it should arguably be the **default**, with root as the opt-in.
- **H5** implicates the **`docs/DEPLOYMENT.md` hand-maintained sha256** approach — it's better than nothing but not a real integrity chain.

## Coverage gaps (NOT certified)

- **Static only** — no daemon was run; the `0644` DB mode, real SSE OOM, and DoS thresholds were not empirically reproduced.
- **Probe command-injection NOT audited** — ~114 probes shell out to `nmap`/`testssl`/`sslyze`/`openssl`/`syft` etc.; argument construction from scanned hostnames/paths needs a **dedicated pass** (candidates this round did not survive verification, but coverage was not exhaustive).
- **XLSX injection blast radius** sampled (sbom/cert/privkey), not all probe evidence keys enumerated.
- **Frozen-binary resource resolution** (`offline_pack`, `sys._MEIPASS`) not reviewed for path traversal / tool substitution.
- **No SCA** — dependency/CVE posture of FastAPI/starlette/openpyxl/weasyprint/sqlalchemy not assessed.
